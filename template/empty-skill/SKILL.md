# {{SKILL_NAME}}

将这个 skill 作为新 Codex skill 的起点模板。

## 这个 Skill 是做什么的

这个 skill 提供了一个最小结构，用来定义：

- 这个 skill 的用途
- 什么时候应该使用它
- 它应该如何处理任务
- 它需要遵守哪些约束

## 什么时候使用

当用户满足以下情况时使用这个 skill：

- 明确按名称要求使用这个 skill
- 需要这个 skill 所代表的工作流帮助
- 需要某个特定领域或任务的可复用操作说明

## 工作流

1. 先澄清用户目标和预期输出。
2. 只收集完成任务所必需的上下文。
3. 应用这个 skill 定义的工作流或领域规则。
4. 按用户期望的格式产出结果。
5. 如有必要，明确说明假设、缺口或后续工作。

## 约束

- 保持说明简洁且具体。
- 避免加载不必要的上下文。
- 当任务比较脆弱时，优先使用确定性步骤。
- 只有在确实能提升可靠性时才添加参考资料或脚本。

## 定制说明

请将这份起始模板替换为以下真实内容：

- 这个 skill 实际覆盖的领域或任务
- 触发使用它的条件
- Codex 应该遵循的具体工作流
- 任何重要的参考资料、脚本或资源文件

如果这个 skill 不再只是一个简短说明，应把详细内容拆到 `references/`、`scripts/` 或 `assets/` 中，并让这个文件继续聚焦在工作流本身。
